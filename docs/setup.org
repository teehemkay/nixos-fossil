#+TITLE: Setup
#+OPTIONS: toc:3

This document covers all setup flows: initial bootstrap of the project, adding a new host, and adding a new fossil repo.

* Initial bootstrap

First-time setup of the project. Run once.

** Prerequisites

- Hetzner Cloud account.
- DigitalOcean account (only when bringing up =secondary-2=).
- Local =nix=, =ssh=, =curl=, =openssl= installed. (=age= and =agenix= are provided by the flake dev shell — run =nix develop=, or =direnv allow= on first use.)
- This repo cloned at =~/dev/my/nixos-fossil=.

** Procedure

*** 1. Register your SSH key as the agenix admin recipient

agenix uses your SSH ed25519 key as both the *recipient* (when encrypting) and the *identity* (when decrypting). This repo's admin key is =~/.ssh/id_ed25519_ext= — a non-default path. agenix only auto-discovers =~/.ssh/id_ed25519= and =~/.ssh/id_rsa=, so every decrypting =agenix= invocation in this guide passes =-i ~/.ssh/id_ed25519_ext=.

Print your SSH public key:

#+begin_src bash
cat ~/.ssh/id_ed25519_ext.pub
#+end_src

Edit =secrets/secrets.nix=, replace the =tmk= placeholder string with the full line (=ssh-ed25519 AAAA... user@host=) from your =id_ed25519_ext.pub=:

#+begin_src bash
$EDITOR secrets/secrets.nix
#+end_src

Back up =~/.ssh/id_ed25519_ext= (private half) to 1Password or equivalent. Losing it means losing decrypt access to every secret in the repo until a host with a recipient key can be reached.

*** Note on the zero-byte placeholders

Steps 2-5 below each populate one of the =secrets/*.age= files. The repo ships these as zero-byte placeholders (committed during implementation so Nix path literals resolve during =nix eval=). =agenix -e <file>= treats any existing =<file>= as encrypted age content and tries to decrypt it before opening the editor — and 0-byte content is not valid age payload, so =agenix= would error.

**Workaround: for the FIRST encryption of each secret, `rm` the placeholder immediately before `agenix -e`.** Once a real encrypted file exists, subsequent rotations use =agenix -e <name>.age -i ~/.ssh/id_ed25519_ext= without the =rm= (the file is now valid encrypted content that agenix must decrypt first). First encryption needs no =-i= — there is nothing to decrypt.

agenix resolves its rules file as =./secrets.nix=, so every =agenix= command below is run from the =secrets/= directory and names the file with no directory prefix. Each step shows the =cd ~/dev/my/nixos-fossil/secrets= + =rm <name>.age && agenix -e <name>.age= form explicitly.

*** 2. Encrypt the fossil-sync password

This password is embedded into HTTPS URLs by =bin/new-repo.sh= and the rotation runbook (=https://syncuser:$PASS@.../...=), so it MUST be URL-safe — no =@=, =:=, =/=, =%=, =?=, =#=, =+=, or whitespace. The command below draws 64 alphanumeric characters straight from =/dev/urandom= — URL-safe, with no dependency beyond coreutils.

Do NOT substitute =openssl rand -base64= or similar — those generate =+/= characters that break URL parsing.

#+begin_src bash
cd ~/dev/my/nixos-fossil/secrets
LC_ALL=C tr -dc 'A-Za-z0-9' < /dev/urandom | head -c 64 > /tmp/sync.pass
rm fossil-sync.age && agenix -e fossil-sync.age
# paste contents of /tmp/sync.pass; one line (vim's trailing newline is
# harmless — bin/new-repo.sh reads the password with $(cat ...), which strips it)
rm /tmp/sync.pass
#+end_src

*** 3. Encrypt tmk-password.age

Generate a yescrypt-hashed password (this is the breaking-glass console password):

#+begin_src bash
cd ~/dev/my/nixos-fossil/secrets
mkpasswd -m yescrypt    # type a strong password; copy the entire $y$...$ hash
rm tmk-password.age && agenix -e tmk-password.age
# paste the hash; one line
#+end_src

*** 4. Create healthchecks.io account + 2 checks

At https://healthchecks.io:

- Create account.
- Create one check per secondary, name them =secondary-1-sync= and =secondary-2-sync=.
- Schedule: =every 5 minutes=, grace =15 minutes=.
- Copy each check's ping URL.

Encrypt the URLs:

#+begin_src bash
cd ~/dev/my/nixos-fossil/secrets
rm healthchecks-secondary-1.age && agenix -e healthchecks-secondary-1.age
# paste URL, one line
rm healthchecks-secondary-2.age && agenix -e healthchecks-secondary-2.age
# paste URL, one line
#+end_src

*** 5. Generate Tailscale auth keys

In Tailscale admin → Keys, create three /tagged/, /one-time/, /pre-approved/, /ephemeral=false/ auth keys; one per host. Tag them e.g. =tag:fossil=.

Encrypt each:

#+begin_src bash
cd ~/dev/my/nixos-fossil/secrets
rm tailscale-authkey-canonical.age && agenix -e tailscale-authkey-canonical.age
# paste auth key
rm tailscale-authkey-secondary-1.age && agenix -e tailscale-authkey-secondary-1.age
rm tailscale-authkey-secondary-2.age && agenix -e tailscale-authkey-secondary-2.age
#+end_src

*** 6. Push to GitHub

#+begin_src bash
git add -A
git commit -m "feat: encrypted secrets initialized"
git remote add origin git@github.com:teehemkay/nixos-fossil.git
git push -u origin main
#+end_src

The repo is **private** — it carries working notes (=journal/=, =todo.org=, =docs/superpowers/=) beyond the deployable config. =system.autoUpgrade= therefore fetches it over SSH using a per-host deploy key; see step 6 of "Adding a host" below.

** Verification

- =nix flake check= surfaces only hardware-config placeholder throws (no other errors).
- =secrets/*.age= all show as encrypted text when =cat='d.
- =git status= clean.

* Adding a host

End-to-end procedure for bringing up a new fossil server.

** Prerequisites

- Initial bootstrap complete (you have the admin keypair and all secrets encrypted).
- Provider (Hetzner Cloud or DigitalOcean) account with an SSH key registered.
- This repo on your laptop, in the right directory.

** Procedure

*** 1. Provision the VM

- Hetzner Cloud: create a CPX11 instance in the target DC, attach the NixOS rescue ISO or boot with the recovery image. Capture the IPv4.
- DigitalOcean: similar.

*** 2. DNS A record

In the =exidia.com= zone (at Hover), add an A record pointing at the VM's IPv4 from step 1:
- =fossil= for canonical
- =fossil1= for secondary-1
- =fossil2= for secondary-2

This record MUST exist before step 6 — ACME's HTTP-01 challenge validates the cert by reaching the host at this name on :80. The explicit record also overrides =exidia.com='s wildcard =*= for that name.

*** 3. First boot — bootstrap install via nixos-anywhere

From your laptop:

#+begin_src bash
nixos-anywhere \
  --flake .#<hostname>-bootstrap \
  --target-host root@<ip> \
  --generate-hardware-config nixos-generate-config \
  ./hosts/<hostname>-hardware.nix
#+end_src

Watch for: kexec → install → reboot. After reboot, =ssh root@<ip>= should work using your SSH key.

*** 4. Capture host SSH host pubkey

#+begin_src bash
ssh root@<ip> 'cat /etc/ssh/ssh_host_ed25519_key.pub'
#+end_src

Copy the output (starts with =ssh-ed25519 AAAA...=).

*** 5. Update secrets/secrets.nix + rekey

In =secrets/secrets.nix=, locate the empty list for this host (one of =canonicalHost=, =secondary1Host=, =secondary2Host=) and replace =[ ]= with a one-element list containing the captured pubkey. For example, after provisioning canonical:

#+begin_src nix
# Before:
canonicalHost = [ ];
# After:
canonicalHost = [ "ssh-ed25519 AAAA...captured...pubkey root@canonical" ];
#+end_src

Then re-encrypt every secret whose recipient list includes this host:

#+begin_src bash
(cd ~/dev/my/nixos-fossil/secrets && agenix --rekey -i ~/.ssh/id_ed25519_ext)
#+end_src

Commit BOTH the rekey AND the generated hardware-config that =nixos-anywhere= produced in step 4. Without committing the hardware-config, =hosts/<hostname>-hardware.nix= in the GitHub repo remains the throwing placeholder, and =system.autoUpgrade= (which pulls from =git+ssh://git@github.com/teehemkay/nixos-fossil?ref=main#<host>=) will fail to evaluate on the next scheduled run, blocking kernel patches:

#+begin_src bash
git add secrets/ hosts/<hostname>-hardware.nix
git commit -m "feat: register $hostname host key + hardware-config"
git push
#+end_src

*** 6. Register the host's deploy key on GitHub

=system.autoUpgrade= fetches the private flake over SSH, authenticating as the host's =ssh_host_ed25519_key=. Register the pubkey captured in step 4 as a *read-only deploy key*:

GitHub → the repo's =Settings= → =Deploy keys= → =Add deploy key=. Title it after the hostname, paste the =ssh-ed25519 AAAA...= line, and leave *Allow write access* unchecked.

Until this is done the host's weekly autoUpgrade cannot fetch the flake. Initial deploys (steps 3 and 7) run from your laptop's local clone, so they are unaffected — only the unattended upgrade needs the deploy key.

*** 7. Promote to full config

#+begin_src bash
nixos-rebuild switch --flake .#<hostname> --target-host root@<ip>
#+end_src

This is the first activation that references agenix secrets. =security.acme= issues the cert. Tailscale joins. fossil-server starts on :443.

*** 8. Smoke test

The argument is the DNS name fossil serves on for that host, NOT the NixOS hostname. Use =fossil.exidia.com= for canonical, =fossil1.exidia.com= for secondary-1, =fossil2.exidia.com= for secondary-2.

#+begin_src bash
bin/smoke-test.sh <dns-name>
# e.g. bin/smoke-test.sh fossil.exidia.com
#+end_src

** Verification

- =bin/smoke-test.sh= all green.
- HTTPS in a browser shows fossil's default page with a valid (non-self-signed) cert.
- =systemctl status fossil-server= on the host shows =active (running)=.
- (Secondaries only) =systemctl list-timers fossil-sync= shows the timer.

** Rollback

If the bootstrap fails partway: re-image the VM and start over from step 3.

If the promote step (#7) fails to activate the new generation: the host stays on the bootstrap config; SSH still works; fix the issue locally and re-run =nixos-rebuild switch=.

* Adding a repo

Cluster-wide creation of a new fossil repo via =bin/new-repo.sh=.

** Prerequisites

- All target hosts deployed and reachable via SSH (configure hostnames in =~/.ssh/config= if needed).
- =/run/agenix/fossil-sync= present and readable on every host (verify with =ssh canonical 'sudo ls -l /run/agenix/fossil-sync'=).

** Procedure

#+begin_src bash
bin/new-repo.sh <reponame>
#+end_src

The script:

1. SSHes to canonical, runs =fossil init=, creates =syncuser= with capability =v=, adds the repo to fossil's =all= list. The sync password is read from =/run/agenix/fossil-sync= on canonical.
2. For each secondary, SSHes in, clones the new repo via =https://syncuser:$PASS@fossil.exidia.com/<name>=, sets =remote-url=, adds to =all=, runs a verifying =fossil all sync -u=.

** Verification

- =bin/new-repo.sh= exits 0.
- Visit =https://fossil.exidia.com/<reponame>/= in a browser — fossil's setup-users page appears.
- After ≤5 min, =https://fossil1.exidia.com/<reponame>/= and =https://fossil2.exidia.com/<reponame>/= serve the same content.

** Post-creation: bootstrap the admin user

Visit =https://fossil.exidia.com/<reponame>/setup_uedit?user=admin= and set a real password for the admin user (the one fossil created during =fossil init= will have the OS username as login).

** Rollback

If =new-repo.sh= fails partway, remove the partially-created repo file from any hosts that got it:

#+begin_src bash
ssh canonical 'sudo rm /var/lib/fossil/museum/<reponame>.fossil'
ssh fossil1.exidia.com 'sudo rm /var/lib/fossil/museum/<reponame>.fossil'
ssh fossil2.exidia.com 'sudo rm /var/lib/fossil/museum/<reponame>.fossil'
#+end_src

Then re-run the script.
