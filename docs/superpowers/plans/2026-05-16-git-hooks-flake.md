# git-hooks.nix Flake Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the repo's pre-commit hook version-controlled, reproducible, and self-installing by defining it declaratively in `flake.nix` via git-hooks.nix.

**Architecture:** Add a `git-hooks` flake input. A new `let` binding `preCommitHooks` (`git-hooks.lib.<devSystem>.run`) defines three hooks — `nixfmt-rfc-style`, `statix`, and a custom `nixos-eval-test`. A new `devShells.aarch64-darwin.default` output consumes `preCommitHooks.shellHook`, which installs `.git/hooks/pre-commit` on dev-shell entry. A committed `.envrc` makes entry automatic for direnv users. No `checks` output is exposed — the eval hook cannot run in a build sandbox.

**Tech Stack:** Nix flakes, [cachix/git-hooks.nix](https://github.com/cachix/git-hooks.nix), direnv, nixfmt-rfc-style, statix.

**Spec:** `docs/superpowers/specs/2026-05-16-git-hooks-flake-design.md` (reviewed, converged 2026-05-16).

## File Structure

| File | Change | Responsibility |
|------|--------|----------------|
| `flake.nix` | Modify | Add `git-hooks` input; add `devSystem`/`pkgs`/`preCommitHooks` `let` bindings; add `devShells` output. |
| `flake.lock` | Modify (auto) | Pins git-hooks.nix and its transitive deps. Updated by `nix flake lock`. |
| `.envrc` | Create | One line, `use flake` — direnv loads the dev shell on `cd`. |
| `.gitignore` | Modify | Ignore `.pre-commit-config.yaml`, generated by the shellHook. |
| `.git/hooks/pre-commit` | Delete | The hand-written hook; replaced by the git-hooks.nix-managed one. Not version-controlled. |
| `CLAUDE.md` | Modify | The "Verifying changes" section — describe the flake-managed hook. |

**Sequencing constraint:** The first `nix develop` (or `direnv allow`) triggers the shellHook, which installs `.git/hooks/pre-commit`. git-hooks.nix refuses to overwrite a foreign hook file. So the hand-written hook MUST be deleted (Task 3) **before** the first dev-shell entry (Task 4). Tasks 1–2 verify the flake with `nix eval`, which evaluates without entering the shell.

### Task 2: Wire the dev shell and pre-commit hooks into flake.nix

**Files:**
- Modify: `flake.nix` (`let` bindings, `outputs` attribute set)

- [ ] **Step 1: Add `devSystem` and `pkgs` `let` bindings**

The `let` block starts:

```nix
    let
      system = "x86_64-linux";

      # Helper: build a full nixosConfiguration for one host.
```

Replace that with:

```nix
    let
      system = "x86_64-linux";

      # Developer machine. The dev shell and its git-hooks are built for this
      # platform. The host configs stay x86_64-linux — they describe remote
      # servers, not this machine.
      devSystem = "aarch64-darwin";
      pkgs = nixpkgs.legacyPackages.${devSystem};

      # Helper: build a full nixosConfiguration for one host.
```

- [ ] **Step 2: Add the `preCommitHooks` binding**

The `let` block ends with the `mkHostEvalTest` helper, immediately before `in`:

```nix
      mkHostEvalTest =
        name:
        nixpkgs.lib.nixosSystem {
          inherit system;
          specialArgs = { inherit inputs; };
          modules = [
            disko.nixosModules.disko
            ./hosts/${name}.nix
            ./hosts/_fixture-hardware.nix
          ];
        };
    in
```

Replace it with:

```nix
      mkHostEvalTest =
        name:
        nixpkgs.lib.nixosSystem {
          inherit system;
          specialArgs = { inherit inputs; };
          modules = [
            disko.nixosModules.disko
            ./hosts/${name}.nix
            ./hosts/_fixture-hardware.nix
          ];
        };

      # Pre-commit hook set, built by git-hooks.nix. Consumed only for its
      # shellHook, which installs .git/hooks/pre-commit on dev-shell entry.
      # Deliberately NOT exposed as a flake `checks` output: the
      # nixos-eval-test hook runs `nix eval`, which needs the nix daemon and
      # cannot run inside a build sandbox.
      preCommitHooks = git-hooks.lib.${devSystem}.run {
        src = ./.;
        hooks = {
          nixfmt-rfc-style.enable = true;
          statix.enable = true;
          nixos-eval-test = {
            enable = true;
            name = "nixos eval-test";
            entry = "${pkgs.writeShellScript "nixos-eval-test" ''
              set -e
              for host in canonical secondary-1 secondary-2; do
                ${pkgs.nix}/bin/nix eval --no-warn-dirty --raw \
                  ".#nixosConfigurations.$host-eval-test.config.system.build.toplevel.drvPath" \
                  >/dev/null
              done
            ''}";
            files = "\\.nix$";
            pass_filenames = false;
          };
        };
      };
    in
```

- [ ] **Step 3: Add the `devShells` output**

The `outputs` attribute set currently is:

```nix
    {
      nixosConfigurations = {
        canonical = mkHost "canonical";
        canonical-bootstrap = mkHostBootstrap "canonical";
        secondary-1 = mkHost "secondary-1";
        secondary-1-bootstrap = mkHostBootstrap "secondary-1";
        secondary-2 = mkHost "secondary-2";
        secondary-2-bootstrap = mkHostBootstrap "secondary-2";
        canonical-eval-test = mkHostEvalTest "canonical";
        secondary-1-eval-test = mkHostEvalTest "secondary-1";
        secondary-2-eval-test = mkHostEvalTest "secondary-2";
      };
    };
```

Replace it with:

```nix
    {
      nixosConfigurations = {
        canonical = mkHost "canonical";
        canonical-bootstrap = mkHostBootstrap "canonical";
        secondary-1 = mkHost "secondary-1";
        secondary-1-bootstrap = mkHostBootstrap "secondary-1";
        secondary-2 = mkHost "secondary-2";
        secondary-2-bootstrap = mkHostBootstrap "secondary-2";
        canonical-eval-test = mkHostEvalTest "canonical";
        secondary-1-eval-test = mkHostEvalTest "secondary-1";
        secondary-2-eval-test = mkHostEvalTest "secondary-2";
      };

      # Dev shell: entering it runs preCommitHooks.shellHook, which installs
      # .git/hooks/pre-commit. Built for the developer's machine (devSystem).
      devShells.${devSystem}.default = pkgs.mkShell {
        inherit (preCommitHooks) shellHook;
      };
    };
```

- [ ] **Step 4: Format `flake.nix`**

Run: `nixfmt flake.nix`
Expected: no output, exit 0.

- [ ] **Step 5: Verify the dev shell evaluates — do NOT enter it yet**

Run: `nix eval --no-warn-dirty --raw '.#devShells.aarch64-darwin.default.drvPath'`
Expected: a single `/nix/store/...-nix-shell.drv` path, exit 0. This evaluates the dev shell and `preCommitHooks` without running the shellHook, so it does not install the hook. (Do not run `nix develop` here — the hand-written hook is still in place and git-hooks.nix would refuse to overwrite it. The dev shell is entered for the first time in Task 4.)

If this errors with an unknown hook option (e.g. `nixfmt-rfc-style` or a custom-hook attribute), consult the git-hooks.nix hook module — `nix flake show github:cachix/git-hooks.nix` and the upstream `modules/` directory list the supported hook names and per-hook options — and adjust before continuing.

- [ ] **Step 6: Commit**

```bash

git commit -m "feat: add git-hooks.nix dev shell with nixfmt, statix, eval-test hooks"
```

Expected: the hand-written hook prints `pre-commit: ok`; the commit succeeds.

### Task 4: Install and verify the managed hook end-to-end

**Files:** none committed — this task verifies behavior.

- [ ] **Step 1: Enter the dev shell to install the hook**

Run: `nix develop --command true`
Expected: git-hooks.nix's shellHook runs and prints a line confirming the hook was installed (e.g. `pre-commit installed at .git/hooks/pre-commit`), exit 0. On first run this also builds git-hooks.nix and may take a few minutes.

Expected: a pre-commit-framework-generated script — it references `pre-commit` / the git-hooks store path, NOT the hand-written `# pre-commit` / `nixfmt --check` script.

Run: `test -f .pre-commit-config.yaml && git status --porcelain .pre-commit-config.yaml`
Expected: `test` exits 0 (the file exists) and `git status --porcelain` prints nothing (the file is gitignored — no untracked-file noise).

- [ ] **Step 3: Verify a misformatted `.nix` file is blocked**

```bash

git add scratch-hook-test.nix
git commit -m "test: this commit must be rejected"
```

Expected: the commit FAILS. The `nixfmt-rfc-style` hook reports `scratch-hook-test.nix` is not formatted. Nothing is committed.

- [ ] **Step 4: Clean up the scratch file**

```bash

rm scratch-hook-test.nix
```

Expected: no output, exit 0. `git status --porcelain` no longer lists `scratch-hook-test.nix`.

Note on the clean-pass path: a clean `.nix` change passing the managed hook is not separately scripted here to avoid committing a throwaway file. It is covered transitively — Tasks 1 and 2 each committed a clean `.nix` change successfully, and the managed hook runs the identical check set (`nixfmt`, `statix`, the three `-eval-test` evaluations).
